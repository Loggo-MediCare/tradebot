## 🔧 技术细节

### 1. 算法对比

**PPO (Proximal Policy Optimization)**:
- 优点: 稳定,易训练
- 缺点: 探索能力有限
- 适合: 离散或简单连续动作

**SAC (Soft Actor-Critic)**:
- 优点: 强探索,样本效率高
- 缺点: 训练时间稍长
- 适合: 连续动作,复杂环境

### 2. 奖励函数演变

**旧版**:
```python
reward = profit / initial_balance
```

**改进版**:
```python
reward = profit + trade_incentive + cash_penalty
```

**高级版**:
```python
reward = (profit +
         risk_penalty +      # 集中度惩罚
         sell_reward +       # 卖出奖励
         auto_trade_reward + # 止损/止盈奖励
         drawdown_penalty)   # 回撤惩罚
```

### 3. 风险管理机制

**止损**:
- 当浮亏 > 10% 时强制卖出
- 保护本金,控制下行风险

**止盈**:
- 当浮盈 > 20% 时卖出 50%
- 锁定利润,降低回撤

**集中度控制**:
- 持股 > 90%: 惩罚 -0.002
- 现金 > 90%: 惩罚 -0.001

**回撤惩罚**:
- 从最高点下跌时额外惩罚
- 鼓励保持稳定增长

## 💡 关键洞察

### 为什么改进版只买不卖?

1. **2024是牛市**: AAPL 全年上涨 35.89%
2. **买入持有最优**: 在单边上涨中,持有是正确的
3. **缺少风险信号**: 没有大幅下跌触发卖出
4. **奖励函数**: 没有足够的卖出激励

### 高级版如何解决?

1. **止盈机制**: 盈利 20% 自动锁定部分利润
2. **卖出奖励**: 每次卖出获得 +0.005 奖励
3. **SAC 探索**: 自动寻找最优买卖时机
4. **回撤控制**: 防止利润大幅回吐
